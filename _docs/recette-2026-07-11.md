## Défauts trouvés et corrigés (au-delà des correctifs de mars)

1. **Crash serveur à chaque nouvel abonné** (P0) : la relation `userCategories`
   injectée sur le modèle User référençait `mappedBy: 'users'` mais le schéma
   `user-category.users` ne déclarait pas `inversedBy` → métadonnées de jointure
   incomplètes → TypeError fatal dans @strapi/database lors de l'émission
   d'événement post-création. Fix : `inversedBy: "userCategories"` dans
   `user-category/schema.json`.
2. **Vérification « auteur bloqué » jamais exécutée** (P1 sécurité) :
   `pagination: { limit: 1 }` faisait planter la requête et le `.catch(() => [])`
   avalait l'erreur — un compte suspendu pouvait commenter. Fix : `limit` à la
   racine + log d'erreur au lieu d'un silence.
3. **PUT settings no-op silencieux** (P1) : le corps enveloppé `{ data: {...} }`
   n'était pas déballé ; Zod strippait la clé inconnue et persistait un objet
   vide (200 OK, réglage perdu). Fix : déballage `body.data ?? body` + GET
   fusionnant defaults + valeurs stockées.

## Diagnostics de mars invalidés

- BUG-5 n'était pas un problème de pagination mais l'appel
  `plugin('users-permissions').service('role').findOne({ type })`, signature
  disparue en v5 → remplacé par `documents(...).findFirst({ filters })`.
- Le correctif `pagination: { page, pageSize }` (BUG-4/BUG-5) est **ignoré
  silencieusement** par le Document Service v5 (vérifié : pageSize=2 renvoyait
  tout). La forme correcte est `limit` / `start` à la racine des params.
